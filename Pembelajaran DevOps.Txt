Git adalah Version Control System (VCS) untuk mengelola perubahan source code

- Repository tempat penyimpanan terpusat untuk seluruh aset dalam sebuah project yang menggunakan Version Code System
GitLab berfungsi sebagai tempat menyimpan source code, mendukung kolaborasi tim, memicu CI/CD pipeline, serta menyediakan backup dan audit trail
- Branch jalur pengembang terpisah dalam sebuah repository
Branch digunakan untuk mengembangkan fitur baru, memperbaiki bug, melakukan eksperimen, dan menjaga stabilitas kode utama
- Commit sebuah rekaman pada saat melakukan perubahan pada repository
Commit berfungsi untuk menyimpan riwayat perubahan, memungkinkan rollback, mendukung kolaborasi tim, menjadi dasar branch & merge, memicu CI/CD, serta mendokumentasikan perubahan kode
- Merge penggabungan Branch yang kita buat
Merge digunakan untuk menggabungkan fitur baru, mengintegrasikan perbaikan bug, menyatukan hasil kerja beberapa developer, dan menjaga branch utama tetap up to date
- Pull Request permintaan untuk penggabungan (merge) perubahan dari satu branch ke branch lain melalui review
Merge Request berfungsi untuk melakukan code review tim, memastikan kode sesuai standar, mendeteksi bug lebih awal, serta mendukung diskusi dan kolaborasi
